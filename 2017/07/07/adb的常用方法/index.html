adb的常用方法

一、常用命令

  1. adb devices
    列举当前连接的调试设备。
  2. adb logcat
    打印log信息。
  3. adb install/uninstall
    安装/卸载apk。
  4. adb pull/push
    调试设备和开发PC之间拷贝文件。
  5. adb start/kill-server
    启动/杀死adb的Server端进程。
  6. adb shell
    进入调试设备的shell界面, 此时可以使用调试设备中的很多指令。
  7. adb connect/disconnect
    通过wifi进行远程连接手机进行调试。
  8. adb shell am
    am即activity manager.
    该命令用来执行一些系统动作, 例如启动指定activity, 结束进程, 发送广播, 更改屏幕属性等. 调试利器.
  9. adb shell pm
    pm即package manager.
    用来执行package相关的操作, 例如安装卸载, 查询系统的安装包等.
  10. adb shell screencap
    截屏, 比截屏快捷键更加方便快捷.（Android4.4及以上系统可用.）
  11. adb shell screenrecord
    录屏, 做demo的话, 可以很方便的用这个命名录制视频, 然后借助工具将其转换成gif图, 超赞. 如下:
  12. adb shell dumpsys
    强大的dump工具, 可以输出很多系统信息. 例如window, activity, task/back stack信息, wifi信息等.

adb端口被占用解决方法

一、杀死占用adb端口的进程

  1. 5037为adb默认端口，输入netstat -ano | findstr “5037”查找哪些进程占用了该端口，记住非0地址的进程的ID。
  2. 打开任务管理器，点击“进程”->“查看”->“选择列”->勾选PID。
  3. 根据进程的ID查找相应的进程，然后结束该进程。
    二、自己配置一个adb端口
  4. 在系统环境变量中增加一个ANDROID_ADB_SERVER_PORT。
  5. 将ANDROID_ADB_SERVER_PORT的值定义为自己想配置的端口号。
